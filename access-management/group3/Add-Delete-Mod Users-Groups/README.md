### **Create Group**

‡∏õ‡πâ‡∏≠‡∏ô‡∏Ñ‡∏≥‡∏™‡∏±‡πà‡∏á `groupadd` ‡πÄ‡∏û‡∏∑‡πà‡∏≠‡∏™‡∏£‡πâ‡∏≤‡∏á‡∏Å‡∏•‡∏∏‡πà‡∏°‡∏ú‡∏π‡πâ‡πÉ‡∏ä‡πâ‡πÉ‡∏´‡∏°‡πà :

 **Options ‡∏ó‡∏µ‡πà‡∏°‡∏µ‡∏≠‡∏¢‡∏π‡πà‡πÉ‡∏ô‡∏Ñ‡∏≥‡∏™‡∏±‡πà‡∏á `groupadd`**

 <table align="center">
	<tr>
		<th>Option</th>
		<th>Description </th>
	</tr>
	<tr>
		<td>-f, ‚Äìforce</td>
		<td>This option forces the command to silently abort if the group with the given name already exists. If used with the -g or ‚Äìgid option and the specified group id already exists, the command forcefully ignores the given group id and assigns a new and unique group id.</td>
	</tr>
  <tr>
		<td>-g GID, ‚Äìgid GID</td>
		<td>This option assigns a specific numeric group id to the newly created group.

  </tr>

</td>
  </tr>
  <tr>
		<td>-o, ‚Äìnon-unique</td>
		<td>Permits adding a group with a non-unique group id (GID).

</td>
  </tr>
  <tr>
		<td>-p PASSWORD, ‚Äìpassword PASSWORD</td>
		<td>Sets an encrypted password for the group.

</td>
  </tr>
</tr>
  <tr>
		<td>-r, ‚Äìsystem</td>
		<td>Creates a system group. System groups have numeric identifiers chosen within the SYS_GID_MIN-SYS_GID_MAX range, as defined in the /etc/login.defs file, instead of GID_MIN and GID_MAX.</td>
  </tr>
<tr>
		<td>-R CHROOT_DIR, ‚Äìroot CHROOT_DIR</td>
		<td>Applies changes in the specified CHROOT_DIR directory and uses the configuration files from that directory.
Useful when managing groups within a chroot environment. </td>
  </tr>
